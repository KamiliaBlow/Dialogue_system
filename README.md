# Диалоговая система
Данный проект представляет Клиент оболочка на (html,js,css) + сервер node.js (c хранилищем ассетов и бд).

Эта версия является бетой. Функции могут меняться и дорабатываться.

Для чего это нужно?
- У меня есть НРИ игры по Sci-Fi тематике (например Lancer RPG), так как я не могу регулярно проводить игровые сессии, чтобы немного занять игроков и познакомить их с будущими НПЦ или поговорить с знакомыми НПЦ была создана эта система.

В этой системе можно:
- Послушать диалог при 1х или более лицах.
- Активно учавствовать в диалоге и выбирать в диалоге ответ, влияющий на дальнейшее развитие диалога.

- Вы так же можете зайти в админ панель, которая показывает, какой пользователь и что выбрал, чтобы на основе выбора строить диалоги дальше.
- В последней версии в замен тягомотному конфигу, был добавлен "Редактор диалогов", который позволяет построить ветвистый диалог.

Чтобы получить админку, используйте endpoint первичной настройки `/api/setup/first-admin` (см. раздел «Безопасность» ниже). Назначение вручную через БД также возможно, но рекомендуется использовать endpoint: он назначает администратором первого пользователя и блокируется, если администратор уже существует.

# Как установить? 

Вам понадобится:
- Node.js v20.11.0+ (сгодится любая версия выше указанной).
- Любой вебсервер, на котором можно разместить клиентскую часть, например: XAMPP, Foundry VTT, Apache и др.

Установка:
1. Содержимое client side скопируйте на сайт в удобное для вас место.
- откройте config.js в папке SCRIPTS и замените REPLACEME:3000 на ваш домен, как пример, это будет выглядеть так: template.ru:3000 .

2. Содержимое server side скопируйте куда угодно на хостируемой машине, запустите CMD и перейдите в папку содержимого server side, в консоли введите:
```
npm install
```

3. Затем откройте config.js и .env внутри папки сервера и вместо REPLACEME вставьте ваш домен, так же не забудьте поменять секрет (REPLACEMESECRETKEY и REPLACEMESETUPKEY).

4. Закиньте ваши ssl сертификаты в папку ssl сервера (Лучше создавать сертификат через certbot, так как скрипт изначально создавался под него). **SSL обязателен в продакшене** — без сертификатов сервер не запустится (если только явно не задать `ALLOW_HTTP=1`, что предназначено только для локальной разработки).

5. Запустите bat файл.

# Безопасность

Конфигурация сервера читается из `server side/.env` (файл **не** должен попадать в git — см. `.gitignore`). Шаблон значений — в `server side/.env.example`. Обязательные параметры:

- `SESSION_SECRET` — сильный случайный ключ (≥32 символа). Сгенерировать: `node -e "console.log(require('crypto').randomBytes(48).toString('base64'))"`
- `SETUP_KEY` — сильный случайный ключ (≥16 символов) для первичной настройки администратора. Сгенерировать: `node -e "console.log(require('crypto').randomBytes(24).toString('base64'))"`
- `CORS_ORIGINS` — список разрешённых доменов через запятую (например: `https://example.com,https://www.example.com`).
- `ALLOW_HTTP` — `1` разрешает запуск без SSL только для локальной разработки (по умолчанию `0`).
- `SESSION_MAX_AGE` — срок жизни сессионной куки в мс (по умолчанию `86400000`, 24 ч).

При дефолтных/пустых секретах или плейсхолдерах **сервер откажется запускаться**.

**Назначение первого администратора:** после регистрации первого пользователя отправьте POST на `/api/setup/first-admin` с `setupKey` из `.env`:
```
curl -X POST https://ваш-домен/api/setup/first-admin \
  -H "Content-Type: application/json" \
  -H "Origin: https://ваш-домен" \
  -d '{"setupKey":"ВАШ_SETUP_KEY"}'
```
Endpoint назначает админом первого пользователя и заблокирован, если администратор уже существует.

Включённые механизмы защиты:
- Security-заголовки (Helmet): CSP, HSTS (при HTTPS), X-Content-Type-Options, frame-ancestors: none и др.
- Rate-limiting: общий на API и строгий на `/api/login`, `/api/register`, `/api/setup/first-admin`.
- Защита от CSRF: SameSite=lax + проверка Origin/Referer для state-changing запросов.
- Пароли хранятся в bcrypt; парольная политика: минимум 8 символов, буквы + цифры.
- Загрузка файлов: проверка расширения, MIME и сигнатур (magic bytes); защита от path traversal для сохранённых путей.

# Как пользоваться?

Вы предоставляете пользователю прямую ссылку до login.html после чего он регистрируется и после запуска (или его пропуска), попадает на экран интерфейса диалогов.

(для админов) прописываете прямую ссылку до admin.html после чего смотрите статистику, взаимодействуете с данными пользователей или переходите в редактор диалогов.

- Работа с базой данных: Так как это самый обычный sqlite он ничем не запаролен, и не имеет сложной защиты, просто берете и подключаетесь программой по типу navicat например.

Я не гарантирую, что данный кусок кода у вас заведется и будет работать, но всегда готова помочь с его настройкой настолько, насколько возможно.
